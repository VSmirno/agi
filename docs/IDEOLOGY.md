# Идеология проекта SNKS AGI

## Ключевой инсайт (Stage 72, 2026-04-07)

Мы обнаружили фундаментальное расхождение между декларируемой архитектурой (самоорганизация, непрерывное обучение, осцилляторная динамика) и реальной реализацией (supervised backprop, batch training, ground truth scaffolding).

### Что пошло не так

Stages 66-71 решали правильную задачу (перейти от символьного входа к пикселям в Crafter) неправильным способом:

- **NearDetector** — supervised classification через near_head (backprop). Это не самоорганизация. Мы обучали нейросеть распознавать объекты "по учебнику", а не из опыта.
- **Semantic GT навигация** — `use_semantic_nav=True` читает ground truth карту мира из движка. Агент "видит" всю карту как бог. Это не перцепция.
- **Controlled env grounding** — visual prototypes собирались в лабораторных условиях (объект размещён рядом, врагов нет). Агент никогда не видел объекты в реальном контексте.
- **Batch training** — собрали данные → обучили → задеплоили → удивились что не работает. Нет петли обратной связи.

Хорошие метрики в экспериментах (smoke 65%, QA 100%) измеряли качество supervised модели, а не способность агента жить в мире. Когда мы попытались сделать демо — агент "дёргался" и не мог собрать ни одного ресурса без читов.

### Правильная архитектура

**Разделение на уровни по аналогии с мозгом:**

| Уровень | Аналогия | Реализация | Обучение |
|---------|----------|------------|----------|
| Зрительная кора (V1/V2) | Сетчатка → features | CNN encoder (frozen) | Предобучен, дообучается редко по триггеру |
| Распознавание объектов | "Это дерево" | ConceptStore.query_visual() — cosine sim | Мгновенное (one-shot grounding из опыта) |
| Уточнение перцепции | "Точнее — берёза" | EMA обновление prototype | Каждая встреча с объектом |
| Каузальная модель мира | "do tree → wood" | ConceptStore causal rules + confidence | Мгновенное (verify ±0.15 на каждом действии) |
| Пространственная память | "Дерево было слева" | CrafterSpatialMap | Каждый шаг |
| Рефлексы | "Zombie = бежать!" | ReactiveCheck (danger only) | One-shot (после первого укуса) |
| Потребности | "Хочу есть" | Drive strengths → ConceptStore.plan() | Непрерывно (физиология) |
| Планирование | "Нужна кирка → нужен стол → нужно дерево" | ChainGenerator backward chaining | Использует каузальную модель |
| Удивление | "Ожидал камень, нашёл пусто" | ConceptStore.record_surprise() | Мгновенное (prediction error) |

### Три скорости обучения

1. **Мгновенное** — один опыт меняет модель мира:
   - Впервые сделал `do` рядом с деревом и получил wood → visual prototype "tree" записан
   - Zombie укусил → confidence "dangerous" = 1.0
   - Предсказал "тут будет камень", оказалось пусто → confidence снижена, surprise записан

2. **Постепенное** — уточнение с каждым опытом:
   - Visual prototype обновляется EMA (0.9 × old + 0.1 × new)
   - Cognitive map растёт с каждым шагом
   - Confidence каузальных правил стабилизируется после десятков подтверждений

3. **Фоновое** — редко, по необходимости:
   - CNN encoder дообучается если perception quality деградирует
   - Это единственный компонент с backprop — и он фоновый, не прерывает жизнь агента

### Непрерывное обучение ≠ batch training

Агент не останавливается чтобы обучиться. Он учится В ПРОЦЕССЕ жизни:
- Каждое действие → verify prediction → update confidence
- Каждый шаг → update spatial map
- Каждая новая встреча → update visual prototype
- Каждое удивление → record surprise

Нет фаз "собрать данные → обучить модель → задеплоить". Модель мира обновляется непрерывно.

### Curriculum через среду, не через training

Обучение управляется не "фазами эксперимента", а тем что агент встречает:

| Фаза | Среда | Что агент осваивает |
|------|-------|---------------------|
| 1 | Без врагов, лес | Находить деревья, собирать wood |
| 2 | Без врагов, лес + горы | Находить stone |
| 3 | Без врагов, полный мир | Coal, iron, crafting |
| 4 | С врагами | Рефлексы выживания, combat |

Переход по достижению метрик. Агент остаётся в фазе пока не освоит.

### Textbook = учитель, не чит

CrafterTextbook (YAML с каузальными правилами) — это как родитель который объясняет ребёнку: "Это дерево. Если ударить — получишь дерево." Правила даны, но визуальное распознавание — только из опыта. Confidence начинается с 0.5 (недоверие) и растёт с каждым подтверждением.

### CNN encoder ≠ наша модель

CNN — это инструмент, как глаз. Он не "думает". Он превращает пиксели в числа (features). Вся "мысль" — в ConceptStore, drives, planning, spatial map, surprise. Эти компоненты работают без backprop: cosine similarity, one-shot запись, confidence update, backward chaining.

Если CNN когда-нибудь заменится на DAF+SKS (осцилляторная перцепция) — вся остальная архитектура не изменится. CNN — заменяемый модуль V1.

## Инсайт: Top-Down Design (Stage 73, 2026-04-07)

### Проблема снизу вверх

Stages 72-73 строились снизу вверх: perception → drives → planning → поведение. Каждый слой содержал хардкод:

- `max(0, 5 - food) * 2.0` — почему 2.0? Потому что программист так решил.
- `if has_sword == 0: drives["wood_sword"] = 3.0` — почему 3.0? Подбирали руками.
- `if took_damage: env.step("do")` — damage reflex, зашитый в код.

Результат: agent не обучается стратегии. 200 эпизодов — survival не растёт. Drives не адаптируются. Agent умирает от зомби 80% случаев, но не учится крафтить меч раньше.

### Правильный подход: сверху вниз

Мысленный эксперимент: представим что мировая модель УЖЕ готова. Как мы сформулируем для неё задачу?

> "Ты играешь в игру. Твоя задача — выживать и собирать ресурсы. У тебя есть параметры (health, food, drink, energy) — если любой дойдёт до 0, ты умрёшь. Есть монстры, от них можно убежать или убить (нужно оружие). Вот справочник крафтов. Вот экран. Вот кнопки."

Вводные для системы:
1. **Цель:** "выживай, собирай ресурсы"
2. **Справочник:** textbook (каузальные правила крафта, боя, выживания)
3. **Экран:** пиксели → visual field (spatial perception из CNN feature map)
4. **Кнопки:** список доступных действий
5. **Тело:** homeostatic переменные (health, food, drink, energy)

ВСЁ. Больше ничего. Система САМА должна вывести:
- Стратегию ("сначала меч, потом кирка")
- Приоритеты ("зомби опаснее голода")
- Рефлексы ("вижу зомби → бей/беги")
- Тактику ("собрать 3 wood → стол → меч")

### Drives = наблюдение за телом

Drives не хардкодятся. Они НАБЛЮДАЮТСЯ:

- Agent видит: "когда рядом zombie, health падает на 2/шаг" → urgency = 2.0
- Agent видит: "food падает на 1 каждые 25 шагов" → urgency = 0.04
- Agent видит: "когда я делаю 'do' рядом с zombie имея sword, zombie исчезает" → решение

Приоритет = скорость падения homeostatic переменной. Не формула программиста, а наблюдение агента за своим телом.

### Стратегия = forward simulation через мировую модель

Agent имеет каузальные правила (textbook + опыт). Он может ПРОМОДЕЛИРОВАТЬ:
- "Если не сделать меч → зомби убьёт → цель провалена (быстро)"
- "Если не поесть → food=0 → умру (через 100 шагов)"
- → меч важнее еды ПРЯМО СЕЙЧАС

Это не RL. Это model-based planning из каузальных правил. World model уже есть — нужно научить agent'а её использовать для принятия решений, а не хардкодить решения за него.

### Правило

**НИКОГДА не хардкодить стратегию, приоритеты или рефлексы. Всё должно вытекать из цели + мировая модель + опыт.**

Если хочется написать `if zombie: fight` — СТОП. Спросить: "может ли agent вывести это сам из textbook + наблюдений?"

## Отложенные вопросы

### DAF/SKS для перцепции

DAF (осцилляторы Курамото/FHN) и SKS (кластеры синхронных осцилляторов) были спроектированы для самоорганизующейся перцепции. Но они работают в другом пространстве (oscillator phases) чем CNN features (z_real vectors). Нужен мост — projection layer CNN→oscillators. Когда этот мост будет построен:

- SKS кластеры заменят ConceptStore.query_visual() — более богатое multi-object perception
- GWS (Global Workspace) заменит простой argmax drives — настоящий winner-take-all attention
- IntrinsicCost (homeostatic firing rate) заменит ручные drive functions

### Правила из удивления

Сейчас surprise только логируется. В будущем: если agent многократно видит неожиданный исход (prediction error) — автоматически создаётся новое каузальное правило. Это discovery — agent узнаёт то что не было в textbook.

### Социальное обучение

Textbook — это односторонняя коммуникация. В будущем: agent может "спрашивать" (генерировать запрос) и получать новые правила. Диалог с учителем.
