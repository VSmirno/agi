# MVP СНКС: Proof-of-Concept

## Цель MVP

Доказать, что архитектура СНКС (Динамические Аттракторные Поля + Семантические Кристаллические Структуры + DCAM-хранение) способна:

1. Формировать устойчивые внутренние представления (СКС) из сырых сенсорных данных
2. Непрерывно обучаться без catastrophic forgetting
3. Предсказывать будущее состояние среды
4. Делать всё это НЕ через backpropagation и НЕ через transformer

**MVP НЕ демонстрирует:** мультимодальность, метакогницию, самосознание, язык, планирование.

---

## Ограничения среды

- **Железо:** мини-ПК, 100 GB RAM, многоядерный CPU (например, AMD Ryzen 9 / Intel i9), опционально GPU (RTX 4090, 24 GB VRAM)
- **ОС:** Linux или Windows
- **Время обучения:** часы-дни, не недели
- **Стек:** Rust (ядро) + Python (эксперименты, визуализация)

## Архитектура MVP

### Что включено

```
+------------------------------------------+
|       Визуальный кодировщик (МИ)         |
|  Вход: 64x64 grayscale изображения       |
|  Выход: SDR-паттерны -> ДАП-активация    |
+------------------------------------------+
            |
            v
+------------------------------------------+
|     ДАП-Ядро (Когнитивное ядро, урезанное)|
|  - Сеть связанных осцилляторов            |

|  - Предсказание следующего входа          |
+------------------------------------------+
            |
            v
+------------------------------------------+
|     DCAM-Хранилище (World Model)          |
|  - HAC-кодирование узлов (D=2048)        |
```

- Глобальное Рабочее Пространство (ГРП)
- Каузальный механизм
- LLM-интерфейс

## Компоненты MVP — детально

### Компонент 1: ДАП-Движок (DAF-Engine)

**Суть:** библиотека для симуляции динамических аттракторных полей.

**Реализация:** Вариант B из концепции — стохастическая динамика на графе:
```
x_i(t+dt) = x_i(t) + dt * [f(x_i) + Σ_j(w_ij * g(x_j - x_i)) + I_i(t)] + √dt * σ * η_i
```

**Параметры для MVP:**
- Число осцилляторов (узлов): **50 000** (начать с 10 000, масштабировать)
- Размерность состояния одного осциллятора: **8** (фаза, амплитуда, частота, 5 внутренних переменных)
- Средняя связность: **50 соседей на узел** (sparse)
- Шаг интегрирования dt: **0.1 мс** (модельного времени)
- Метод интегрирования: **Euler-Maruyama** (для стохастических ODE)
- Шум σ: настраиваемый параметр

**Правила обучения (локальные, без backprop):**
- STDP (Spike-Timing Dependent Plasticity): усиление связей между синхронизирующимися узлами
- Гомеостатическая пластичность: поддержание средней активности в целевом диапазоне
- Структурная пластичность: добавление связей между часто коактивирующимися узлами, удаление слабых связей

**Потребление ресурсов (оценка):**
- Состояние узлов: 50 000 × 8 × 4 байта = 1.6 MB
- Связи: 50 000 × 50 × (4 + 4 + 4) байта (вес + задержка + тип) = 30 MB
- Буфер истории (для STDP): ~100 MB
- Итого ДАП: **~150 MB RAM**

**Требования к производительности:**
- Минимум 1000 шагов симуляции в секунду на CPU (без GPU)
- С GPU: минимум 10 000 шагов в секунду

**Критерий готовности:**
- [ ] Система стабильно работает без расхождения и без затухания
- [ ] Формируются устойчивые аттракторы при постоянном входном сигнале
- [ ] Аттракторы различимы для разных входных сигналов
- [ ] STDP обучение изменяет аттракторный ландшафт

### Компонент 2: Визуальный кодировщик

**Суть:** преобразование изображений 64×64 grayscale в ДАП-совместимый формат.

**Архитектура:**
1. **Фиксированный слой Габор-фильтров** (не обучаемый):
   - 8 ориентаций × 4 масштаба × 4 фазы = 128 фильтров
   - Применяются к входу, генерируют 128 карт признаков
2. **SDR-кодирование:**
   - Каждая карта признаков → скалярный кодировщик → SDR (спарсификация через k-WTA: top-k активаций = 1, остальные = 0)
   - Итоговый SDR: ~4096 бит, спарсность ~2-5%
3. **SDR → ДАП-вход:**
   - Активные биты SDR подаются как внешние токи I_i(t) на соответствующие узлы ДАП

**Потребление ресурсов:**
- Фильтры Габора: фиксированные, ~1 MB
- SDR: 4096 бит = 512 байт на кадр
- Итого: **пренебрежимо мало**

**Критерий готовности:**
- [ ] Похожие изображения генерируют SDR с высоким overlap
- [ ] Различные изображения генерируют SDR с низким overlap
- [ ] SDR стабилен при небольших возмущениях входа (шум, сдвиг на 1-2 пикселя)

### Компонент 3: Формирование СКС

**Суть:** самоорганизация устойчивых паттернов в ДАП-сети при повторяющихся стимулах.

**Механизм:**
1. При подаче стимула X группа узлов начинает синхронизироваться
2. STDP усиливает связи внутри синхронизированной группы
3. При повторной подаче X та же группа активируется быстрее и сильнее
4. Через N повторений группа становится устойчивым аттрактором = СКС

**Метрики СКС:**
- **Идентификация:** кластеризация активных узлов при подаче стимула (алгоритм: spectral clustering или DBSCAN по фазовой синхронизации)
- **Стабильность:** корреляция между паттернами при повторной подаче одного стимула
- **Разделимость:** расстояние между паттернами разных стимулов
- **Целевые значения:** стабильность > 0.9, разделимость > 0.7 (косинусное расстояние)

**Потребление ресурсов:** входит в бюджет ДАП-Движка.

**Критерий готовности:**
- [ ] При подаче 10 различных геометрических фигур формируются 10 различимых СКС
- [ ] СКС стабильны: повторная подача того же стимула активирует ту же группу узлов (корреляция > 0.85)
- [ ] Похожие стимулы (круг/эллипс) имеют частично перекрывающиеся СКС
- [ ] Непохожие стимулы (круг/квадрат) имеют минимально перекрывающиеся СКС

---

### Компонент 4: Предсказательный механизм

**Суть:** система предсказывает следующий входной стимул на основе выученных последовательностей.

**Механизм:**
1. Подаётся последовательность стимулов: A → B → C → A → B → C → ...
2. Каузальные связи формируются: СКС_A → СКС_B → СКС_C → СКС_A
3. При активации СКС_A система предактивирует СКС_B (prediction)
4. Если приходит B — prediction error ≈ 0, обучение минимально
5. Если приходит D — prediction error >> 0, обучение усиливается

**Реализация prediction error:**
```
PE_i(t) = |predicted_state_i(t) - actual_state_i(t)|
learning_rate_i(t) = base_rate * (1 + α * PE_i(t))
```

**Потребление ресурсов:** входит в бюджет ДАП-Движка (каузальные связи — подмножество связей графа).

**Критерий готовности:**
- [ ] Точность предсказания > 80% для выученных последовательностей длины 3-5
- [ ] Prediction error снижается экспоненциально при обучении
- [ ] При нарушении последовательности генерируется всплеск prediction error
- [ ] Система обучается быстрее HTM (NuPIC) на тех же последовательностях

### Компонент 5: DCAM-Хранилище

**Суть:** персистентное хранение World Model в формате Dual-Code Attractor Memory.

**Параметры для MVP:**
- HAC: D = **2048** (вместо 4096 в полной версии — экономия памяти)
- Максимум узлов: **100 000**
- SSG: 4 слоя (структурный, каузальный, темпоральный, модуляторный)
- LSH-индекс: 32 хеш-таблицы × 16 бит = 32 бита на ключ
- Эпизодический буфер: **10 000 эпизодов**

**Потребление ресурсов:**
- HAC-коды узлов: 100 000 × 2048 × 4 байта = **800 MB**
- SSG граф: 100 000 × 50 связей × 48 байт = **240 MB**
- LSH-индекс: **~100 MB**
- Эпизодический буфер: 10 000 × ~20 KB = **200 MB**
- Итого DCAM: **~1.5 GB RAM**

**Консолидация (аналог сна):**
- Запускается по расписанию (каждые N циклов) или по заполнению эпизодического буфера
- Переигрывает эпизоды из буфера, укрепляет частые паттерны в SSG
- Удаляет устаревшие/дублирующие эпизоды

**Критерий готовности:**
- [ ] Записанные СКС восстанавливаются из хранилища с точностью > 0.95
- [ ] Семантический поиск (по содержанию) работает за O(1) через LSH
- [ ] Консолидация перемещает частые паттерны из эпизодической в семантическую память
- [ ] Система переживает перезагрузку без потери знаний

### Компонент 6: Визуализация и мониторинг

**Суть:** веб-интерфейс для наблюдения за работой системы в реальном времени.

**Минимальный набор:**
- Тепловая карта активности ДАП-сети (2D проекция через t-SNE/UMAP)
- Графики: prediction error, число сформированных СКС, средняя активность
- Список текущих СКС с их "силой" и составом
- Визуализация текущего входа и предсказания
- Граф каузальных связей между СКС

**Стек:** Python (FastAPI + WebSocket) + JavaScript (D3.js или Three.js)

**Потребление ресурсов:** ~500 MB RAM (буфер визуализации).

## Датасет для MVP

### Уровень 1: Синтетические геометрические фигуры
- 10 классов: круг, квадрат, треугольник, эллипс, прямоугольник, пятиугольник, звезда, крест, ромб, стрелка
- Вариации: размер (3 градации), положение (случайное), поворот (0-360°), шум (3 уровня)
- Итого: ~10 000 уникальных изображений 64×64 grayscale
- Подача: потоком, с повторениями (каждое изображение показывается 10-50 раз)

### Уровень 2: Последовательности
- Детерминированные последовательности: A→B→C→A→B→C...
- Стохастические: A→B(70%)/C(30%)→D→A...
- С переключением контекста: первые 1000 кадров — одна последовательность, потом другая

### Уровень 3 (stretch goal): MNIST
- 28×28 → resize до 64×64
- 70 000 рукописных цифр
- Цель: формирование 10 СКС (по одной на цифру) + промежуточные СКС для общих признаков

## Эксперименты для валидации

### Эксперимент 1: Формирование СКС
- **Вход:** 10 геометрических фигур, каждая показывается 50 раз в случайном порядке
- **Ожидание:** формируются 10 устойчивых СКС
- **Метрика:** Normalized Mutual Information (NMI) между кластерами активности и истинными классами > 0.8
- **Сравнение:** SOM (Self-Organizing Map) на тех же данных

### Эксперимент 2: Непрерывное обучение
- **Фаза A:** обучение на фигурах 1-5 (500 показов)
- **Фаза B:** обучение на фигурах 6-10 (500 показов)
- **Тест:** подать фигуры 1-5 → проверить, что СКС сохранились
- **Метрика:** точность распознавания фигур 1-5 после Фазы B > 85%
- **Сравнение:** MLP + SGD (должен показать catastrophic forgetting)

### Эксперимент 3: Предсказание последовательностей
- **Вход:** детерминированные последовательности длины 3, 5, 7
- **Метрика:** точность предсказания следующего элемента после обучения
- **Сравнение:** HTM (NuPIC), simple RNN
- **Ожидание:** СНКС сравнима с HTM, превосходит простой RNN на коротких последовательностях

### Эксперимент 4: Устойчивость к шуму
- **Вход:** изученные фигуры с добавлением гауссова шума (σ = 0.1, 0.2, 0.3)
- **Метрика:** точность активации правильной СКС при зашумлённом входе
- **Ожидание:** graceful degradation (не обрыв, а плавное снижение точности)

### Эксперимент 5: Персистентность DCAM
- Обучить систему, сохранить DCAM на диск, перезагрузить
- **Метрика:** точность распознавания после перезагрузки = точности до перезагрузки (±1%)
- **Ожидание:** полная восстановимость

## Критерии успеха MVP

### Must-have (без этого MVP провален)
1. СКС формируются из сырых визуальных данных (NMI > 0.7)
2. Непрерывное обучение без catastrophic forgetting (удержание > 80%)
3. Система работает без backpropagation и без transformer

### Should-have (сильно укрепляют доказательство концепции)
4. Предсказание последовательностей работает (точность > 70%)
5. World Model персистентна (DCAM работает)
6. Производительность позволяет обучение за часы, не дни

### Nice-to-have (бонус)
7. Работает на MNIST (NMI > 0.6 без supervised learning)
8. Превосходит SOM/HTM хотя бы по одной метрике
9. Визуализация в реальном времени

## Риски MVP и митигации

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| ДАП не сходится к стабильным аттракторам | Высокая | Начать с проверенных моделей (Курамото); тщательный подбор параметров; гомеостаз |
| STDP недостаточно для формирования СКС | Средняя | Добавить BCM-правило (Bienenstock-Cooper-Munro); попробовать Oja's rule |
| Производительность 50K узлов слишком низкая на CPU | Средняя | Оптимизировать через SIMD/vectorization в Rust; перейти на GPU раньше |
| SDR-кодирование теряет слишком много информации | Низкая | Увеличить размерность SDR; попробовать multi-scale кодирование |
| Консолидация DCAM нестабильна | Средняя | Начать с простого: копирование часто активных паттернов без replay |

## Что MVP докажет (и что нет)

### Докажет:
- ДАП-динамика может формировать устойчивые представления из сырых данных
- Локальные правила обучения (STDP + гомеостаз) достаточны для самоорганизации
- DCAM формат пригоден для хранения и восстановления World Model
- Архитектура не требует backpropagation

### Не докажет (это задача следующих этапов):
- Масштабируемость до миллионов узлов
- Мультимодальную интеграцию
- Каузальное рассуждение
- Планирование и agency
- Что это приближает к AGI (только первый шаг)
