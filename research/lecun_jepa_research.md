## 1. Ключевые идеи ЛеКуна и JEPA

### 1.1 "A Path Towards Autonomous Machine Intelligence" (2022)

Опорная работа ЛеКуна [LeCun, 2022, OpenReview] представляет архитектурный план для достижения human-level AI. Это не статья с экспериментами, а **архитектурный манифест** -- подробная спецификация того, как должна быть устроена система, обладающая автономным интеллектом.

**Центральный тезис:** Для достижения автономного интеллекта машинного уровня необходимы:

1. **World Model** -- внутренняя модель мира, способная предсказывать последствия действий
2. **Обучение в латентном пространстве** -- предсказание абстрактных представлений, а не сырых сенсорных данных
3. **Иерархическое планирование** -- многоуровневое планирование с разными временными масштабами
4. **Intrinsic motivation** -- внутренняя мотивация, не зависящая от внешних наград

### 1.2 Архитектура когнитивной системы по ЛеКуну

ЛеКун предлагает систему из 6 взаимосвязанных модулей:

```
┌─────────────────────────────────────────────────────────┐
│                    Configurator                          │
│  (модулирует все остальные модули в зависимости от      │
│   задачи, настраивает параметры, субцели)               │
├─────────────┬────────────────────────┬──────────────────┤
│             │                        │                  │
│   Perception│     World Model        │    Actor         │
│   Module    │  ┌──────────────────┐  │    Module        │
│             │  │ Предсказание     │  │                  │
│  x → s(x)  │  │ следующего       │  │  a = policy(s)   │
│             │  │ латентного       │  │                  │
│             │  │ состояния        │  │                  │
│             │  └──────────────────┘  │                  │
├─────────────┴────────────────────────┴──────────────────┤
│                Short-Term Memory                         │
│  (хранит историю состояний, предсказаний, действий)     │
├─────────────────────────────────────────────────────────┤
│                   Cost Module                            │
│  (intrinsic cost = discomfort, curiosity,               │
│   extrinsic cost = task-specific)                       │
└─────────────────────────────────────────────────────────┘
```

**Ключевые модули:**

1. **Perception Module (Enc):** Кодирует наблюдение x в латентное представление s = Enc(x). Критически важно: кодировщик должен быть **информационно неполным** -- он отбрасывает нерелевантную информацию, сохраняя только то, что полезно для предсказания.

2. **World Model:** Ядро системы. Предсказывает следующее латентное состояние:
   ```
   s_{t+1} = Predictor(s_t, a_t, z)
   ```
   где z -- латентная переменная, захватывающая неопределённость (стохастичность мира). Предсказание происходит в **латентном пространстве**, а НЕ в пространстве пикселей.

3. **Cost Module:** Вычисляет "стоимость" (энергию) состояний:
   - **Intrinsic cost:** Аналог гомеостаза, любопытства, дискомфорта
   - **Extrinsic cost:** Задачно-зависимые цели
   - Действует как **energy function**, а не как reward signal

4. **Actor Module:** Оптимизирует последовательность действий для минимизации cost:
   ```
   a* = argmin_a Sum_t C(s_t)  (через world model)
   ```
   Это **планирование через модель**, не trial-and-error RL.

5. **Short-Term Memory:** Хранит контекст -- последовательность (s, a, cost) для reasoning и планирования.

6. **Configurator:** Мета-уровень: выбирает, какие подмодули использовать, устанавливает субцели, модулирует attention. Функционально аналогичен исполнительным функциям префронтальной коры.

### 1.3 Joint Embedding Predictive Architecture (JEPA)

JEPA -- конкретная реализация идеи обучения world model. Вместо генеративного подхода (предсказать все пиксели) JEPA предсказывает **абстрактные представления**.

**Фундаментальная идея:**

```
     x ──────→ [Encoder] ──→ s_x
                                    ↘
                                     Compare(s_x, s_y_pred)
                                    ↗
     y ──────→ [Target Encoder] ──→ s_y
                     ↑
              [Predictor] ──→ s_y_pred
                     ↑
                    s_x
```

Два ключевых отличия от генеративных моделей:

1. **Предсказание в латентном пространстве.** Предсказывается не y (пиксели), а s_y (представление y). Это позволяет предиктору игнорировать непредсказуемые детали (шум текстур, точное освещение).

2. **Joint embedding.** И вход x, и цель y проходят через энкодеры, которые отображают их в общее латентное пространство. Предиктор работает только в этом пространстве.

**Зачем предсказывать представления, а не пиксели:**

ЛеКун приводит аргумент информационной сложности: видеокадр содержит огромное количество информации, большая часть которой непредсказуема (движение листьев на ветру, точные блики). Генеративная модель вынуждена моделировать ВСЮ эту информацию. JEPA может выучить представление, отбрасывающее непредсказуемое, и предсказывать только релевантное.

**Проблема коллапса и её решение:**

Главная опасность joint embedding -- **representation collapse**: тривиальное решение, где энкодер отображает все входы в одну точку (s_x = s_y = const). Тогда предсказание тривиально, но бесполезно.

Методы предотвращения коллапса:
- **Contrastive methods** (SimCLR, MoCo): негативные пары. ЛеКун считает их неэффективными -- требуют тщательного подбора негативных примеров.
- **Non-contrastive methods** -- предпочтительный подход ЛеКуна:
  - **VICReg** (Variance-Invariance-Covariance Regularization): регуляризация, поддерживающая дисперсию и декорреляцию представлений
  - **Barlow Twins**: минимизация избыточности
  - **EMA (Exponential Moving Average)** для target encoder: target encoder обновляется медленнее, чем основной -- это создаёт "самодистилляцию"

### 1.4 Иерархическая JEPA (H-JEPA)

ЛеКун предлагает **стек JEPA на разных уровнях абстракции**:

```
Уровень 3 (план дня):        JEPA-3: предсказание на часы вперёд
                                  ↕
Уровень 2 (тактика):         JEPA-2: предсказание на минуты
                                  ↕
Уровень 1 (моторика):        JEPA-1: предсказание на миллисекунды
                                  ↕
Сенсорный вход              [Encoder]
```

На каждом уровне JEPA предсказывает в своём латентном пространстве. Более высокие уровни:
- Оперируют более абстрактными представлениями
- Предсказывают на более длительные горизонты
- Обновляются медленнее

Это соответствует нейробиологической иерархии кортекса: V1 обрабатывает миллисекунды, префронтальная кора -- часы/дни.

### 1.5 Energy-Based Models (EBM)

JEPA -- частный случай **Energy-Based Models**. EBM определяют совместимость (x, y) через энергетическую функцию:

```
E(x, y) : низкая энергия = совместимы, высокая = несовместимы
```

В JEPA энергия определяется как расстояние между предсказанным и реальным представлением:

```
E(x, y) = || Predictor(Enc(x)) - TargetEnc(y) ||^2
```

Преимущество EBM перед генеративными моделями: EBM не нужно моделировать полное распределение p(y|x). Достаточно определить, какие y совместимы с x (низкая энергия), а какие нет.

## 2. Детальный разбор работ

### 2.1 I-JEPA (Image Joint Embedding Predictive Architecture)

**Статья:** Assran et al., "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", CVPR 2023.

**Идея:** Самообучение на изображениях БЕЗ аугментаций данных (data augmentations), что отличает I-JEPA от всех contrastive/non-contrastive методов (SimCLR, DINO, MAE).

**Архитектура:**

```
Изображение ──→ Разбиение на патчи ──→ Масштабируемый ViT-encoder
                                              │
                    ┌─────────────────────────┤
                    │                         │
              Context patches           Target patches (маскированные)
                    │                         │
                    ↓                         ↓
              [Context Encoder]       [Target Encoder (EMA)]
                    │                         │
                    ↓                         ↓
              [Predictor]   ──→   Predicted target repr.
                                         ↕ (loss)
                                  Actual target repr.
```

**Ключевые решения:**
1. **Маскирование в пространстве патчей** (не пикселей). Маскируются целые регионы изображения.
2. **Предсказание в латентном пространстве.** Предиктор предсказывает не пиксели замаскированных патчей, а их представления.
3. **Multi-block masking.** Маскируется несколько несвязных регионов -- это заставляет модель формировать семантические представления (нельзя "экстраполировать текстуру").
4. **Target encoder -- EMA-копия** context encoder-а (как в BYOL/DINO).
5. **Без аугментаций.** Не нужны crop, flip, color jitter. Это упрощает применение к не-визуальным модальностям.

**Результаты:**
- На ImageNet-1K: конкурентоспособен с DINO v2, MAE, при значительно меньшем количестве вычислений
- Линейное зондирование (linear probing) показывает семантически богатые представления
- Работает хорошо на low-shot задачах (1% данных)
- **Без аугментаций** -- это принципиально важно для обобщения на другие модальности

**Значимость для СНКС:** I-JEPA показывает, что предсказание в латентном пространстве работает на практике и формирует семантические представления без внешней разметки.

### 2.2 V-JEPA (Video Joint Embedding Predictive Architecture)

**Статья:** Bardes et al., "Revisiting Feature Prediction for Learning Visual Representations from Video", 2024.

**Идея:** Расширение JEPA на видео. Маскируются пространственно-временные блоки, предсказываются представления замаскированных фрагментов.

**Архитектура:**
- Входное видео разбивается на пространственно-временные "тубы" (spatio-temporal tubes)
- Маскирование: удаляются целые пространственно-временные регионы (до 90% видео)
- Предиктор должен восстановить представления замаскированных регионов, используя контекст

**Ключевые находки:**
1. Видео содержит **временную структуру**, которая заставляет модель выучивать более глубокие семантические представления, чем на статичных изображениях.
2. V-JEPA **без fine-tuning** показывает state-of-the-art на задачах распознавания действий.
3. Латентные представления V-JEPA захватывают **каузальную структуру** действий (что происходит до/после).
4. Модель обучается предсказывать на разных временных масштабах одновременно.

**Значимость для СНКС:** V-JEPA демонстрирует, что предсказание в латентном пространстве масштабируется до временных последовательностей и захватывает каузальную/темпоральную структуру -- это напрямую перекликается с предсказательным движком СНКС (PredictionEngine).

### 2.3 MC-JEPA (Multimodal Contrastive JEPA)

MC-JEPA и его варианты исследуют мультимодальное обучение через JEPA-фреймворк: объединение визуальных, аудио и текстовых представлений в общем латентном пространстве.

Основная идея: каждая модальность имеет свой энкодер, но предиктор учится переводить представления одной модальности в представления другой. Это создаёт **cross-modal prediction** -- предсказание визуального представления по аудио, и наоборот.

**Значимость для СНКС:** Прямая связь с Этапом 7 (текстовая модальность) и cross-modal experiments. JEPA предлагает элегантный способ мультимодальной интеграции через общее латентное пространство.

### 2.4 Latent Variable Models и роль z

Критически важный элемент JEPA, часто упускаемый: **латентная переменная z**.

```
s_y_pred = Predictor(s_x, z)
```

Переменная z захватывает **неопределённость** -- всё то, что нельзя предсказать из x. ЛеКун подчёркивает: мир стохастичен, и предиктор не должен пытаться предсказать одно конкретное будущее. Вместо этого z параметризует **множество возможных будущих**.

Это фундаментальное отличие от:
- **Generative models** (VAE, diffusion): моделируют полное распределение p(y|x)
- **Discriminative models**: дают одно предсказание
- **JEPA**: определяет **множество совместимых** будущих через набор z, при этом энергия остаётся низкой

## 3. Критика LLM от ЛеКуна

### 3.1 "Dark Matter of Intelligence"

ЛеКун использует метафору "тёмной материи интеллекта": то, что LLM не могут выучить -- это аналог тёмной материи в физике. Большая часть интеллекта невидима в текстовых данных.

**Аргументация:**

1. **Ребёнок за первые 4 года** получает ~50 петабайт визуальной информации. Вся совокупность текстов, на которых обучаются LLM -- ничтожная доля этого.

2. **Текст -- это верхушка айсберга.** Текст описывает только малую часть знаний о мире. Большинство нашего понимания (физика объектов, пространственные отношения, интуитивная биология) никогда не вербализуется.

3. **"Тип 1" знаний** (Канеман): быстрые, интуитивные, перцептивные решения -- это 99% когнитивных операций. LLM могут имитировать только "Тип 2" (медленное, символическое рассуждение), и то плохо.

### 3.2 Почему авторегрессивное предсказание токенов недостаточно

ЛеКун аргументирует по нескольким направлениям:

1. **Дискретность vs непрерывность.** Мир непрерывен. Токенизация -- грубая аппроксимация. Особенно проблематично для физического мира, который требует непрерывных представлений.

2. **Экспоненциальная ошибка.** При авторегрессивной генерации ошибки накапливаются экспоненциально. Каждый следующий токен генерируется на основе предыдущих (включая ошибочные). В латентном пространстве JEPA эта проблема менее остра, т.к. представления более абстрактны и робастны.

3. **Модальная бедность текста.** Текст не содержит grounding. "Стол" -- это набор статистических ассоциаций, а не трёхмерный объект с массой, текстурой, тактильными свойствами.

4. **Отсутствие world model.** Авторегрессивная модель -- это не модель мира. Это модель текста О мире. Это принципиальная разница.

### 3.3 Необходимость World Models

ЛеКун настаивает: путь к AGI лежит через World Models, а не через масштабирование LLM.

**World Model по ЛеКуну:**
- Позволяет предсказывать последствия действий
- Работает в абстрактном пространстве (не пиксели, не токены)
- Поддерживает иерархическое планирование
- Обучается из сенсорного опыта (не текста)
- Может "мечтать" -- симулировать будущее без реальных действий

## 4. Связь с нейронауками

### 4.1 Predictive Coding: Фристон vs ЛеКун

| Аспект | Фристон (Active Inference) | ЛеКун (JEPA) |
|--------|---------------------------|---------------|
| Математика | Variational free energy, Bayesian inference | Energy-based models, gradient descent |
| Предсказание | Генеративное (предсказать сенсорный вход) | В латентном пространстве (абстрактное) |
| Обучение | Минимизация surprise (свободной энергии) | Минимизация prediction error в latent space |
| Действие | Active inference: действовать, чтобы подтвердить предсказания | Планирование через world model |
| Иерархия | Иерархические генеративные модели | Иерархические JEPA |
| Стохастичность | Prior preferences + precision weighting | Латентная переменная z |
| Реализуемость | Вычислительно тяжело, мало реализаций | Продемонстрировано на ImageNet, видео |

**Ключевое различие:** Фристон предлагает **генеративный** подход (предсказать полный сенсорный вход), ЛеКун -- **дискриминативный** в латентном пространстве (предсказать абстрактное представление). ЛеКун аргументирует, что генеративный подход слишком расточителен: зачем предсказывать каждый пиксель, если большинство из них непредсказуемы?

**Точки конвергенции:**
- Оба считают prediction error центральным вычислительным принципом
- Оба признают необходимость иерархии
- Оба включают механизм intrinsic motivation
- Оба отвергают чистый RL как путь к AGI

### 4.2 Биологические корреляты JEPA

1. **Predictive coding в кортексе.** Эмпирически подтверждено, что кортикальные колонки генерируют предсказания и вычисляют prediction error (Rao & Ballard, 1999). JEPA формализует это для латентных представлений.

2. **Mismatch negativity (MMN).** Нейрофизиологический сигнал, возникающий при нарушении предсказания. Аналог prediction error в JEPA.

3. **Hippocampal replay.** Гиппокамп "проигрывает" прошлый опыт во сне -- аналог world model simulation в JEPA.

4. **Dorsal/ventral streams.** Два потока обработки зрительной информации (что/где) -- аналог разных уровней JEPA с разными абстракциями.

5. **Default Mode Network (DMN).** Активен при "мечтании" -- аналог world model simulation без сенсорного входа.

## 5. Сравнение JEPA vs СНКС

### 5.1 Таблица сравнения

| Аспект | JEPA (ЛеКун) | СНКС (текущее состояние) |
|--------|-------------|--------------------------|
| **Фундамент** | Energy-based models, gradient-based optimization | Динамические аттракторные поля, FHN-осцилляторы |
| **Представления** | Латентные вектора (выход ViT/encoder) | СКС (кластеры синхронных осцилляторов) |
| **Обучение** | Backpropagation (через loss в latent space) | STDP + гомеостаз (локальные правила) |
| **Предсказание** | Predictor в латентном пространстве | PredictionEngine (cluster-space transitions) |
| **Память** | Short-term memory (буфер состояний) | DCAM (HAC + SSG + LSH), EpisodicBuffer |
| **World Model** | JEPA IS the world model | CausalWorldModel + DcamWorldModel |
| **Модальности** | Визуальная (I-JEPA), видео (V-JEPA), мультимодальная | Визуальная + текстовая (Stage 7) |
| **Метакогниция** | Configurator (мета-модуль) | GWS + MetacogMonitor + policies (Stage 8) |
| **Мотивация** | Intrinsic cost module | Curiosity (Exp 9), prediction error modulation |
| **Forgetting** | Не адресовано явно | Retention > 100% (Exp 2) |
| **Биологичность** | Средняя (использует backprop) | Высокая (STDP, FHN, no backprop) |
| **Масштабируемость** | Доказана на ImageNet, видео | Доказана на shapes, MNIST |
| **Открытость стохастичности** | Латентная переменная z | Шум Euler-Maruyama (sigma_noise) |

### 5.2 Глубинные сходства

1. **Предсказание как центральный механизм.** И JEPA, и СНКС строят свою архитектуру вокруг предсказания. В СНКС: PredictionEngine (Exp 3) предсказывает следующий кластер по текущему. В JEPA: Predictor предсказывает следующее латентное представление.

2. **Rejection of pixel-level prediction.** СНКС уже работает в абстрактном пространстве: предсказание происходит на уровне cluster IDs / SKS, а не пикселей. Это совпадает с философией JEPA.

3. **Intrinsic motivation.** СНКС (Exp 9, curiosity) использует prediction error для мотивации исследования, как предлагает ЛеКун.

4. **Модель мира.** CausalWorldModel в СНКС -- это прямой аналог World Model в архитектуре ЛеКуна.

5. **Отвержение чистого RL.** И СНКС, и ЛеКун отвергают trial-and-error RL. СНКС использует модель мира для планирования (ментальная симуляция, Exp 8).

6. **Мета-уровень.** GWS + MetacogMonitor в СНКС функционально аналогичны Configurator в архитектуре ЛеКуна.

### 5.3 Принципиальные различия

1. **Backpropagation.** JEPA использует backpropagation для обучения энкодеров и предиктора. СНКС принципиально его отвергает, полагаясь на STDP. Это **фундаментальное** расхождение.

2. **Тип представлений.** JEPA: непрерывные вектора в латентном пространстве нейросети. СНКС: паттерны синхронных осцилляций (СКС). СКС -- это **динамические** аттракторы, а не статические вектора.

3. **Проблема коллапса.** JEPA борется с representation collapse через VICReg/EMA. СНКС не имеет этой проблемы в том же виде, т.к. СКС -- это устойчивые паттерны динамической системы, которые по определению различны (разные аттракторные бассейны).

4. **Масштаб демонстрации.** JEPA доказана на ImageNet/Kinetics (миллионы примеров). СНКС -- на MNIST и синтетических shapes. Разрыв значительный.

5. **Темпоральная обработка.** JEPA (особенно V-JEPA) обрабатывает время через пространственно-временные патчи. СНКС обрабатывает время **нативно** через динамику осцилляторов -- время встроено в саму физику системы.

6. **Латентная переменная z.** В JEPA z явно параметризует неопределённость будущего. В СНКС стохастичность встроена через шум (sigma_noise), но нет явной параметризации множества возможных будущих.

## 6. Предложения для этапов 9+

На основе анализа JEPA выделяю конкретные идеи, которые могут быть интегрированы в СНКС:

### 6.1 Этап 9: "Предсказание в пространстве СКС" (SKS-Space Prediction Enhancement)

**Мотивация (из JEPA):** Текущий PredictionEngine работает на уровне cluster ID transitions. Это грубый дискретный предиктор. JEPA показывает, что предсказание в **непрерывном** латентном пространстве значительно мощнее.

**Предложение:** Создать непрерывное латентное пространство СКС, где каждый СКС представлен не только как набор узлов, но как **вектор** (например, средний firing rate vector кластера, или HAC-вектор). Тогда предсказание можно делать в этом непрерывном пространстве.

**Конкретные шаги:**
1. Определить SKS embedding: каждому СКС сопоставить вектор в R^d (через firing rate statistics, или через HAC-кодирование)
2. Заменить дискретный transition predictor на предсказание в непрерывном SKS-embedding space
3. Использовать **STDP-совместимый** механизм для обучения предиктора (не backprop)
4. Метрика: улучшение accuracy предсказания (Exp 3) и новый эксперимент на предсказание с разными горизонтами

**Связь с debt:** Exp 18 (BroadcastPolicy) можно интегрировать сюда, т.к. broadcast в GWS определяется предсказательной мощью -- более точные предсказания получают приоритет.

**Эксперименты:**
- Exp 19: SKS embedding quality -- NMI между nearest neighbors в embedding space и ground truth
- Exp 20: Continuous prediction accuracy vs discrete (baseline Exp 3)

### 6.2 Этап 10: "Иерархическое предсказание" (Hierarchical Prediction, вдохновлено H-JEPA)

**Мотивация (из JEPA):** ЛеКун предлагает иерархию JEPA с разными временными масштабами. У СНКС нет многоуровневого предсказания.

**Предложение:** Создать два (или более) уровня предсказания:
1. **Низкий уровень (L1):** предсказание следующего СКС в последовательности (текущий PredictionEngine, улучшенный в Stage 9)
2. **Высокий уровень (L2):** предсказание "цели" -- какой СКС (или группа СКС) будет активен через N шагов

**Конкретные шаги:**
1. Определить **мета-СКС** -- кластеры СКС (группы часто со-активных СКС). Это аналог более высокого уровня абстракции.
2. L2-предиктор работает с мета-СКС и предсказывает на более длинном горизонте
3. L2 модулирует L1: предсказание на L1 учитывает "цель" от L2 (аналог top-down influence в predictive coding)
4. Обучение через STDP на разных временных шкалах (быстрая STDP для L1, медленная для L2)

**Эксперименты:**
- Exp 21: Формирование мета-СКС (кластеризация СКС-последовательностей)
- Exp 22: Иерархическое предсказание -- accuracy на длинном горизонте L2 vs flat predictor

### 6.3 Этап 11: "Ментальная симуляция с множественными будущими" (Multi-Future Simulation, вдохновлено латентной z)

**Мотивация (из JEPA):** Латентная переменная z позволяет JEPA представлять МНОЖЕСТВО возможных будущих. Текущая ментальная симуляция в СНКС (Exp 8) детерминистична -- предсказывает одно будущее.

**Предложение:** Расширить ментальную симуляцию для поддержки множественных сценариев:
1. При планировании (ментальной симуляции) генерировать N разных траекторий через варьирование начальных условий / шума
2. Оценивать каждую траекторию через cost function (аналог Cost Module ЛеКуна)
3. Выбирать действие, минимизирующее ожидаемый cost по всем сценариям

**Конкретные шаги:**
1. Модифицировать mental simulation для поддержки стохастических траекторий (варьировать sigma_noise)
2. Добавить "cost function" -- обобщение confidence из MetacogMonitor
3. Planning = min_{action} E_z[cost(trajectory(action, z))]
4. Интегрировать с GWS: результаты планирования broadcast-ятся через GWS

**Эксперименты:**
- Exp 23: Multi-trajectory planning vs single-trajectory -- improvement в accuracy / efficiency на задаче навигации (расширение Exp 8)
- Exp 24: Robustness к стохастичности среды

### 6.4 Этап 12: "Intrinsic Cost Module" (вдохновлено Cost Module ЛеКуна)

**Мотивация:** ЛеКун отделяет Cost Module от World Model. В СНКС curiosity (Exp 9) -- единственная intrinsic motivation, и она интегрирована ad hoc.

**Предложение:** Выделить отдельный модуль оценки "стоимости" состояний:
1. **Homeostatic cost:** штраф за отклонение от "нормы" (аналог гомеостаза, уже частично есть в homeostasis.py)
2. **Epistemic cost:** отрицательный -- снижает стоимость для состояний с высокой неопределённостью (любопытство, расширение Exp 9)
3. **Goal cost:** задачно-зависимый, модулируется Configurator/MetacogMonitor
4. Общий cost = weighted sum, веса модулируются метакогнитивным уровнем

**Эксперименты:**
- Exp 25: Cost-driven exploration vs curiosity-only (Exp 9 baseline)
- Exp 26: Goal-directed behavior через cost shaping

### 6.5 Этап 13: "Configurator" (Meta-Control, расширение Stage 8 GWS)

**Мотивация:** ЛеКун выделяет Configurator как критический модуль, модулирующий все остальные. В СНКС GWS + MetacogMonitor -- начало этого, но нет механизма **динамической реконфигурации** системы.

**Предложение:**
1. Configurator как расширение MetacogMonitor: может менять не только политику (NoisePolicy), но и параметры системы:
   - Какой уровень иерархии активен
   - Параметры STDP (скорость обучения)
   - Порог детекции СКС
   - Горизонт предсказания
2. Configurator учится через мета-обучение: какие конфигурации приводят к успеху в разных контекстах

**Эксперименты:**
- Exp 27: Adaptive configuration vs fixed -- improvement на mixed-task benchmark
- Exp 28: Transfer learning через Configurator adaptation

### 6.6 Сводный роадмап

```
Stage 9:  SKS-Space Prediction Enhancement        [debt cleanup + JEPA prediction]
Stage 10: Hierarchical Prediction (H-JEPA)         [multi-scale temporal]
Stage 11: Multi-Future Mental Simulation           [latent z, planning]
Stage 12: Intrinsic Cost Module                     [unified motivation]
Stage 13: Configurator / Meta-Control               [dynamic reconfiguration]
```

Зависимости:
```
Stage 8 (done) ──→ Stage 9 (prediction) ──→ Stage 10 (hierarchy)
                         │                        │
                         └──→ Stage 11 (multi-future) ←──┘
                                    │
                                    ↓
                              Stage 12 (cost) ──→ Stage 13 (configurator)
```

## 7. Риски и противоречия

### 7.1 Фундаментальное противоречие: backpropagation

JEPA в своих реализациях (I-JEPA, V-JEPA) полностью полагается на backpropagation. СНКС принципиально его отвергает. Это создаёт **основной риск**: идеи JEPA, работающие с backprop, могут не работать со STDP.

**Митигация:**
- Заимствовать **архитектурные идеи** (иерархия, prediction в latent space, multiple futures), НЕ механизмы обучения
- Для каждой заимствованной идеи проверять: существует ли STDP-совместимый аналог?
- Литература по predictive coding с Hebbian learning существует (Rao & Ballard, 1999; Whittington & Bogacz, 2017) -- это мост между JEPA и STDP

### 7.2 Проблема representation collapse для СНКС

В JEPA collapse предотвращается через VICReg/EMA. В СНКС аналогичная проблема: **degeneracy** -- все осцилляторы синхронизируются в один гигантский кластер.

**Текущее решение:** Гомеостаз (homeostasis.py) предотвращает это. Но при масштабировании до иерархических представлений (Stage 10) проблема может вернуться на более высоком уровне.

**Митигация:** Явно тестировать на collapse/degeneracy при каждом расширении. Возможно, потребуется "гомеостаз для мета-СКС".

### 7.3 Масштабируемость

JEPA показана на миллионах изображений. СНКС -- на сотнях/тысячах. Предложенные этапы 9-13 увеличат сложность системы. Нужно убедиться, что базовые механизмы масштабируются.

**Митигация:**
- Каждый этап должен включать scaling test (аналог Exp 1)
- Использовать AMD 92GB для тяжёлых экспериментов
- Stage 9 -- это ключевая проверка: если SKS embedding не масштабируется, дальнейшие этапы под вопросом

### 7.4 Сложность интеграции

Добавление 5 новых этапов с новыми модулями создаёт риск "франкенштейна" -- несвязных компонентов. JEPA элегантна именно своей минимальностью.

**Митигация:**
- Каждый этап должен быть инкрементальным расширением, а не новым модулем
- Stage 9 модифицирует PredictionEngine (не добавляет новый)
- Stage 10 расширяет Stage 9 (не параллельная система)
- Принцип СНКС "один механизм, разные масштабы" должен соблюдаться

### 7.5 Отсутствие явной латентной z в СНКС

JEPA явно моделирует неопределённость через z. В СНКС стохастичность -- через шум осцилляторов. Это может быть **недостаточно** для планирования множественных будущих.

**Митигация:**
- Stage 11 (Multi-Future) требует исследования: достаточно ли варьирования sigma_noise для генерации разнообразных траекторий?
- Возможно, потребуется явная параметризация неопределённости (но через осцилляторный механизм, не backprop)

### 7.6 Риск "чужеродной прививки"

JEPA и СНКС построены на разных философских основаниях:
- JEPA: минимизация энергии через оптимизацию (optimization-first)
- СНКС: самоорганизация через динамику (dynamics-first)

Бездумное заимствование может разрушить внутреннюю когерентность СНКС.

**Митигация:**
- Для каждой идеи задавать вопрос: "Можно ли это реализовать через динамику осцилляторов?"
- Если нет -- либо найти динамический аналог, либо отказаться
- Приоритет: **архитектурные принципы** (иерархия, multi-scale, prediction in abstract space) переносимы. **Механизмы обучения** (backprop, EMA, VICReg) -- нет.

## 8. Итоговые рекомендации

### Что заимствовать из JEPA:

1. **Предсказание в непрерывном абстрактном пространстве** -- Stage 9. Самая прямая и ценная идея. СНКС уже работает с абстрактными представлениями (СКС), но предсказание на уровне дискретных ID теряет информацию.

2. **Иерархия временных масштабов** -- Stage 10. У СНКС есть все предпосылки (осцилляторы естественно имеют разные частоты), но нет формальной иерархии предсказания.

3. **Множественные будущие** -- Stage 11. Стохастичность СНКС (шум) -- готовая основа, нужно её "оседлать" для планирования.

4. **Unified cost/energy function** -- Stage 12. Обобщение curiosity + confidence + homeostasis в единый "энергетический ландшафт".

5. **Configurator как мета-уровень** -- Stage 13. Расширение GWS + MetacogMonitor до полноценного meta-control.

### Чего НЕ заимствовать:

1. **Backpropagation.** Ни в каком виде. СНКС -- STDP-only.
2. **ViT/Transformer-based encoder.** У СНКС свой encoder (Gabor + SDR), и это принципиально.
3. **EMA для target encoder.** Нет аналога в STDP-парадигме. Вместо этого -- стабильность аттракторов.
4. **VICReg/Barlow Twins.** Регуляризация через loss -- это optimization-first. Вместо этого -- гомеостаз.

### Ключевой вывод

JEPA и СНКС удивительно близки на уровне **архитектурных принципов** (prediction-centric, abstract representations, world model, hierarchical, intrinsic motivation) и диаметрально противоположны на уровне **механизмов** (backprop vs STDP, optimization vs self-organization). Это означает, что архитектурные идеи ЛеКуна можно использовать как **дорожную карту**, но каждую идею нужно **переизобретать** в терминах осцилляторной динамики.
