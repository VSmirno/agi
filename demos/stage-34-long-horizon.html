How It Works

The HierarchicalPlanner decomposes high-level goals into a 3-level tree:


Level 2 — Strategic: "What subgoals are needed?" Uses backward chaining from the CausalWorldModel. For a DoorKey task: get_key → unlock_door → reach_goal.


Level 1 — Tactical: "How to achieve each subgoal?" Matches Skills from the SkillLibrary. Navigate to key, execute pickup_key skill, navigate to door, execute toggle_door skill.


Level 0 — Primitive: "Which buttons to press?" Concrete actions: forward, turn_left, turn_right, pickup, toggle.


When execution deviates from the expected state (e.g., key drops), the planner detects the deviation and creates a recovery plan from the actual state. This re-planning costs only 1.67x overhead.
