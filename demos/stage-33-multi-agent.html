      How It Works

      In СНКС, agents think in concepts (SKS — stable knowledge structures), not in words. When Agent A discovers that "picking up a key changes state from key_present to key_held", this is encoded as a CausalLink: action=pickup, context={key_present} → effect={key_held}.


      Agent A sends this knowledge to Agent B via a ConceptMessage — a structured data packet containing SKS IDs, HAC embeddings (2048-dim vectors), and causal links. No natural language is involved.


      Agent B integrates the received causal links into its own world model using confidence-weighted merging. Now Agent B can predict what happens when it picks up a key — without ever having done it.


      This is fundamentally different from LLM-based multi-agent systems where agents exchange text prompts. In СНКС, communication is at the semantic level — exchanging meaning, not symbols.
